### 🚀 Projects

**AI-Governed CMS Infrastructure & Multi-Layered Perimeter Defense**

![Focus](https://img.shields.io/badge/Focus-AI_Security_%26_Infrastructure-007ec6?style=flat-square)
![Status](https://img.shields.io/badge/Status-Live_%26_Autonomous-22863a?style=flat-square)
![Version](https://img.shields.io/badge/SentinelNode-v1.0-blueviolet?style=flat-square)

> "Implementing Governed Cognition to secure public-facing assets through Defense-in-Depth and Autonomous Threat Analysis"

* **The Goal:** Deploy a high-performance, self-monitoring WordPress environment on a **Hetzner VPS**, utilizing a "Defense-in-Depth" architecture and the **Governed Cognition** framework to satisfy strict availability and zero-trust integrity requirements.
* **The Stack:** Hardened **Ubuntu Pro** (LTS security patching), **OpenLiteSpeed** with Native WAF, **CrowdSec** (Global Threat Intelligence), **Python 3**, **SQLite**, and the **Google Gemini API**.
* **Security Controls Implemented:**
  * **Edge Governance & Traffic Shaping:** Developed extensive **Cloudflare WAF** custom rules to enforce geo-fencing, mitigate Layer 7 bot attacks, and protect administrative endpoints before traffic reaches the origin server.
  * **Active Host-Based Access Control:** Implemented a **"Default-Deny"** firewall policy via **UFW**, securing SSH via key-based authentication. Replaced legacy Fail2ban with **CrowdSec**, leveraging a real-time, peer-to-peer threat intelligence network to automatically null-route malicious actors at the network layer.
  * **Cryptographic File Integrity Monitoring (FIM):** Engineered **SentinelNode 1.0**, a custom Python daemon operating in an isolated virtual environment **outside WordPress itself** — architecturally immune to plugin-layer compromise. Running via automated cron jobs, it implements a dual-verification model: **MD5** hashes for WordPress core files validated against the live **WordPress.org upstream cryptographic API** (immutable external truth), and **SHA-256** hashes for sensitive configs (`wp-config.php`, `.htaccess`) validated against a **`chattr +i` immutable local baseline** — ensuring no single point of baseline trust.
  * **NIST AU-3 Compliant Audit Trail:** Every scan is recorded to a **WAL-journaled SQLite** database with full forensic provenance: UUID `trace_id`, millisecond `ts_ms` timestamp, file path, hashing algorithm, local and trusted hash values, anomaly flag, and action taken — satisfying non-repudiation requirements for chain-of-custody integrity.
  * **Autonomous AI Forensics (The Alethic Doctor):** Integrated the **Google Gemini API** into the SentinelNode pipeline via a **Cascading Model Discovery** architecture — automatically detecting and prioritizing the highest available Gemini model at runtime with graceful degradation. Built on Emil Uzelac's published **Governed Cognition** framework ([*"From Fragile Glue to Governed Cognition"*](https://github.com/emiluzelac/governed-cognition)), the LLM acts as a **secondary reasoning engine** bound by strict architectural laws: a **deterministic SHA-256 gate** (math fires before AI), a **mass-infection circuit breaker** (>5 simultaneous anomalies bypasses AI entirely), a **stale evidence watchdog** (alerts if the cron daemon misses a cycle), and a `temperature=0.1` epistemic constraint — enabling real-time forensic analysis of obfuscated zero-day payloads without relying on traditional malware signatures. Kernel-governed agents in the original study achieved **zero unsafe actions** across 1,200 controlled episodes vs. 39–43% unsafe action rates in ungoverned architectures.
  * **Adaptive Intelligence Engine:** Implements a **per-file clean streak tracker** that logs consecutive verified scans to detect behavioral drift over time, surfacing auto-hardening candidates at the 30-scan clean threshold.
  * **Real-Time SecOps Alerting:** Engineered a **Telegram** bot integration to instantly push cryptographically-traced alerts and AI-generated forensic reports to mobile. Designed on a **silence-as-signal** principle — alerts fire exclusively on anomalies, eliminating notification fatigue while guaranteeing zero-latency breach awareness.
  * **Roadmap — Automated Remediation (Hard & Soft Healing):** "Hard Healing" will instantly roll back tampered core files by re-fetching pristine versions directly from **WordPress.org** upon breach detection. "Soft Healing" will grant the **Alethic Doctor** governed write-access to surgically excise injected backdoors from dynamic assets while preserving legitimate developer changes.
